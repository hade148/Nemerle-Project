# תרגיל 5 – Jack Compiler (Project 11) בשפת Nemerle

## מה הפרויקט הזה עושה?

מטרת הפרויקט היא לפתח מהדר (Compiler) שלם עבור שפת התכנות **Jack** (שפה מונחית עצמים הדומה ל-Java המשמשת בקורס Nand to Tetris). המהדר מתרגם קבצי קוד מקור (`.jack`) לקוד מכונה וירטואלית (`.vm`) הניתן להרצה על גבי ה-VM Emulator של פלטפורמת Nand to Tetris.

מהדר זה מהווה המשך ישיר למנתח התחבירי (Syntax Analyzer) שפותח בתרגיל 4 (Project 10), כאשר מבנה המנתח הוסב לפולט קוד VM בעזרת מודולים ייעודיים:
1. **Symbol Table (טבלת סמלים)**: לניהול שמות המשתנים, סוגיהם (types), הקטגוריות שלהם (field, static, local, arg) והאינדקסים הריצה שלהם בשני טווחי השפעה (Class scope ו-Subroutine scope).
2. **VM Writer (כותב ה-VM)**: מודול עזר ליצירת פקודות ה-VM השונות (push, pop, arithmetic commands, labels, branching, functions, calls, returns).
3. **Compilation Engine**: המנוע העיקרי המבצע ניתוח תחבירי רקורסיבי (Recursive Descent Parsing) ויוצר באופן ישיר את פקודות ה-VM התואמות לכל מבנה תחבירי.

הקומפיילר מטפל בכל רכיבי השפה כולל:
- משתני מחלקה (static, field) ומשתנים מקומיים/ארגומנטים.
- הצהרות על מחלקות, קונסטרקטורים, מתודות ופונקציות סטטיות.
- מבני בקרה: `if-else`, `while`.
- ביטויים אריתמטיים ולוגיים מורכבים (כולל שימוש במחלקות ה-OS כמו `Math.multiply` ו-`Math.divide`).
- עבודה עם מערכים (`let a[i] = x`), ומחרוזות (string literals).

## תוכניות הבדיקה וההרצה ב-VM Emulator

הקומפיילר נבדק ואומת בהצלחה על כל שש תוכניות הבדיקה הרשמיות של פרק 11 ב-Nand to Tetris:

1. **Seven**: מבצעת חישוב אריתמטי פשוט `1 + (2 * 3)` ומדפיסה את התוצאה (`7`) בפינה השמאלית העליונה של המסך.
2. **ConvertToBin**: מבצעת המרה של מספר דצימלי ממיקום `RAM[8000]` לייצוג בינארי במיקומים `RAM[8001]..RAM[8016]`.
3. **Square**: משחק אינטראקטיבי המאפשר להזיז ריבוע שחור על המסך, להגדיל/להקטין אותו ולצאת. בודק קונסטרקטורים, מתודות, קריאות למתודות וסגמנט `this`.
4. **Average**: קולט רשימת מספרים מהמשתמש, שומר אותם במערך, מחשב את הממוצע ומדפיס את התוצאה. בודק שימוש במערכים ובמחרוזות.
5. **Pong**: משחק פונג מלא ואינטראקטיבי הבודק אובייקטים מרובים, קשרי גומלין ביניהם ומשתנים סטטיים.
6. **ComplexArrays**: מבצע חמישה חישובים מורכבים במערכים (כולל השמות רקורסיביות וקריאות פונקציות בתוך גישה לאינדקס מערך) ומשווה את התוצאה לצפי.

כל התוכניות עברו קומפילציה מושלמת, וקוד ה-VM שנוצר נטען והורץ בהצלחה מלאה בתוך ה-**VM Emulator** ללא שגיאות ובדיוק לפי הדרישות המצופות.
